## 📊 SEMANTIC VALENCE GROUPS
For the automatic segmentation of knowledge and access, the Utyansky System utilizes the following Namespaces:

| Namespace | Description | Applicability |
| :--- | :--- | :--- |
| `universal` | Standards, GOSTs, general norms. | All systems |
| `design_core` | Styles, ergonomics, visual rules. | UI/UX Modules |
| `legal_tech` | Legislation, regulations, legal norms. | Legal Department |
| `media_prod` | Prompts, generation rules, media logic. | Media Studios |
| `industrial` | Industrial standards, KPIs, reporting. | Factory Systems |
| `vision_ai` | Computer vision and AI analysis parameters. | AI Analytics |
| `secure_vault` | Secret protocols and segmentation keys. | Security Shield |

## 📦 BUNDLED GROUPING PRINCIPLE
Modules can request access to sets of valences (e.g., the "Builder" pack includes `universal` + `design_core`). This ensures data flexibility and security "out of the box."
