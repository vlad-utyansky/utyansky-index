## 📊 ГРУППЫ СЕМАНТИЧЕСКОЙ ВАЛЕНТНОСТИ
Для автоматической сегментации знаний и доступа, Система Утянского использует следующие пространства имён:

| Namespace | Описание | Применимость |
| :--- | :--- | :--- |
| `universal` | Стандарты, ГОСТы, общие нормы. | Всем системам |
| `design_core` | Стили, эргономика, визуальные правила. | UI/UX Модули |
| `legal_tech` | Законодательство, регламенты, нормы права. | Юр-Департамент |
| `media_prod` | Промпты, правила генерации, медиа-логика. | Медиа-Студии |
| `industrial` | Промышленные стандарты, KPI, отчетность. | Заводские системы |
| `vision_ai` | Параметры компьютерного зрения и AI-анализа. | AI-Аналитика |
| `secure_vault` | Секретные протоколы и ключи сегментации. | Security-Щит |

## 📦 ПРИНЦИП ПАКЕТНОЙ ГРУППИРОВКИ
Модули могут запрашивать доступ к наборам валентностей (Напр. Пакет "Builder" включает `universal` + `design_core`). Это обеспечивает гибкость и безопасность данных "из коробки".
